# PDF-RAG

Offline, vectorless document question-answering. Upload PDFs (or plain
text/Markdown), get answers with page-accurate citations, powered by SQLite
FTS5 + BM25 lexical retrieval, a hierarchical document-node structure
(chapter → section → subsection → paragraph), and either fast extractive
answering or an optional local Ollama model for generated answers.

**Status:** backend test suite is green — 89/89 tests passing
(`backend/tests`). See `docs/baseline_2026-07-11.md` for the before/after
record of the bug-fixing pass that got it there.

## Requirements

| Requirement | Version |
| --- | --- |
| Python | 3.12 or 3.13 |
| pip or uv | Any recent version |
| Ollama | Optional, for generated (non-extractive) answers |

## Setup

```powershell
cd D:\PDF-RAG
uv venv .venv --python 3.12
.venv\Scripts\Activate.ps1
uv pip install -r requirements.txt

# Required: the spaCy English model isn't installable via pip alone
python -m spacy download en_core_web_sm
```

(Swap `uv pip install` for plain `pip install` if you don't use uv.)

Install OCR support for scanned PDFs when needed:

```powershell
uv pip install -e ".[pdf]"
```

## Running the API server

```powershell
.venv\Scripts\python.exe -m uvicorn backend.app.api.main:app --reload
```

The server starts on `http://127.0.0.1:8000` by default. Interactive API
docs are auto-generated by FastAPI at `http://127.0.0.1:8000/docs`.

**Full endpoint reference for frontend development: [`docs/apis.md`](docs/apis.md).**

## Ollama (optional)

Ollama is only used for answer *synthesis*; retrieval is always lexical
(FTS5/BM25), never vector-based.

```powershell
ollama pull qwen3.5:4b
$env:RAG_USE_OLLAMA = "1"
$env:RAG_ACTIVE_MODEL = "qwen3.5:4b"
```

Without Ollama, the backend answers using fast extractive matching
(pattern-based fact extraction + fuzzy sentence/passage matching) — no
LLM call, no GPU required.

## Environment variables

All settings are read from `RAG_`-prefixed environment variables (see
`backend/app/core/config.py`). Every one below has a working default —
you only need to set the ones you want to change.

```powershell
$env:RAG_DATA_DIR = "backend/data"              # documents/, indexes/, knowledge/, trees/
$env:RAG_SQLITE_PATH = "metadata.sqlite3"        # relative to RAG_DATA_DIR, or ":memory:"
$env:RAG_OLLAMA_BASE_URL = "http://localhost:11434"
$env:RAG_ACTIVE_MODEL = "qwen3.5:4b"
$env:RAG_GENERATION_MODEL = "llama3.2"
$env:RAG_SPARSE_TOP_K = "40"
$env:RAG_FINAL_CONTEXT_CHUNKS = "8"
$env:RAG_TEMPERATURE = "0.1"
$env:RAG_USE_OLLAMA = "0"                        # "1" to enable generated answers
$env:RAG_FORCE_OCR = "0"
$env:RAG_USE_TREE_SEARCH = "1"
$env:RAG_DEBUG_MODE = "0"
$env:RAG_BACKEND_HOST = "127.0.0.1"
$env:RAG_BACKEND_PORT = "8000"
$env:RAG_FRONTEND_ORIGIN = "http://localhost:3000"   # CORS-allowed origin
$env:RAG_MAX_UPLOAD_SIZE_MB = "100"
```

## Notes

The application is vectorless. Retrieval uses hierarchical document nodes,
SQLite FTS5, heading and phrase indexes, and BM25 (via `bm25s`) — no
embeddings, no vector database. The removed desktop and dense-retrieval
configuration is recorded in `CHANGELOG.md`.
