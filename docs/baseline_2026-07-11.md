## Bug fixed before baseline could even be measured

`backend/app/core/config.py`: `Settings.data_dir` was typed `DirectoryPath`, which
requires the directory to exist at validation time. But that directory is created by
`ensure_data_dirs(settings)`, which needs a constructed `Settings` first. Net effect:
`get_settings()` — called directly by many tests and by `RagServiceV2` — raised
`ValidationError` on any fresh checkout where `backend/data` doesn't exist yet. Fixed by
using a plain `Path` and calling `ensure_data_dirs` inside `get_settings()` itself so the
directories always exist by the time any caller gets a `Settings` object.
